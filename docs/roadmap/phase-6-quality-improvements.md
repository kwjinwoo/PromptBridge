# Phase 6 — Quality Improvements

## Objective

Improve transformation quality, compatibility, observability, and maintainability
after the core product value is demonstrated.

## Candidate scope

- User glossary and domain presets
- Markdown AST and selective Tree-sitter analysis
- Application-specific behavior
- Model-specific prompt tuning
- Expanded regression and compatibility suites
- Streaming preview
- Privacy-preserving optional history

## Exit criteria

Phase 6 is intentionally open-ended. Each selected improvement requires its own
accepted scope, measurable success criterion, and evidence. Candidate features
are not commitments until promoted through a decision or roadmap update.
